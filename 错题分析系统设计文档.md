# 软考错题分析系统设计方案

## 一、项目概述

### 1.1 项目目标
帮助用户通过浏览器插件采集软考网站的错题，进行多维度统计分析，实现基于错题的高效学习，最终通过考试。

### 1.2 核心功能
1. **错题采集**：通过浏览器插件自动/手动采集错题
2. **数据存储**：后端API接收并存储错题数据
3. **统计分析**：多维度分析错题，生成学习洞察
4. **学习复习**：提供错题练习和复习功能
5. **进度追踪**：追踪学习进度和效果

## 三、数据模型

### 3.1 错题表 (wrong_questions)
```sql
CREATE TABLE wrong_questions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    question_id TEXT NOT NULL,        -- 原始网站题目ID
    question TEXT NOT NULL,           -- 题目内容
    options TEXT,                     -- 选项JSON
    user_answer TEXT NOT NULL,        -- 用户答案
    correct_answer TEXT NOT NULL,     -- 正确答案
    analysis TEXT,                    -- 解析内容
    category TEXT,                    -- 题目分类
    chapter TEXT,                     -- 章节
    source_url TEXT,                  -- 来源URL
    is_mastered BOOLEAN DEFAULT 0,    -- 是否已掌握
    review_count INTEGER DEFAULT 0,   -- 复习次数
    last_review_time DATETIME,        -- 最后复习时间
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 3.2 学习记录表 (study_records)
```sql
CREATE TABLE study_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    question_id INTEGER,
    action_type TEXT NOT NULL,        -- 'practice', 'review', 'mastered'
    result BOOLEAN,                   -- 是否正确
    time_spent INTEGER,               -- 用时（秒）
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (question_id) REFERENCES wrong_questions (id)
);
```

### 3.3 每日学习表 (daily_stats)
```sql
CREATE TABLE daily_stats (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    date DATE NOT NULL UNIQUE,
    total_questions INTEGER DEFAULT 0,
    correct_count INTEGER DEFAULT 0,
    wrong_count INTEGER DEFAULT 0,
    practiced_count INTEGER DEFAULT 0,
    mastered_count INTEGER DEFAULT 0
);
```

## 六、核心算法

### 6.1 遗忘曲线复习
基于艾宾浩斯遗忘曲线，实现智能复习提醒：

```javascript
const reviewIntervals = [1, 3, 7, 14, 30, 60]; // 天

function getNextReviewInterval(correctCount, wrongCount) {
    const masteryLevel = correctCount / (correctCount + wrongCount);

    if (masteryLevel >= 0.9) {
        return reviewIntervals[5]; // 60天
    } else if (masteryLevel >= 0.7) {
        return reviewIntervals[4]; // 30天
    } else if (masteryLevel >= 0.5) {
        return reviewIntervals[3]; // 14天
    } else if (masteryLevel >= 0.3) {
        return reviewIntervals[2]; // 7天
    } else if (masteryLevel >= 0.1) {
        return reviewIntervals[1]; // 3天
    } else {
        return reviewIntervals[0]; // 1天
    }
}
```

### 6.2 薄弱点检测
通过分析错题关键词，识别薄弱知识点：

```javascript
function detectWeakPoints(questions) {
    const keywordCount = {};

    questions.forEach(q => {
        const keywords = extractKeywords(q.question);
        keywords.forEach(kw => {
            if (!keywordCount[kw]) {
                keywordCount[kw] = { total: 0, wrong: 0 };
            }
            keywordCount[kw].total++;
            if (q.user_answer !== q.correct_answer) {
                keywordCount[kw].wrong++;
            }
        });
    });

    return Object.entries(keywordCount)
        .filter(([kw, data]) => data.wrong / data.total < 0.5)
        .sort((a, b) => a[1].wrong - b[1].wrong)
        .slice(0, 10);
}
```

### 6.3 练习推荐
根据用户薄弱点和复习周期，智能推荐练习题目：

```javascript
function recommendQuestions(questions, limit = 10) {
    const now = Date.now();

    // 筛选需要复习的题目
    const dueQuestions = questions.filter(q => {
        if (!q.last_review_time) return true;
        const daysSinceReview = (now - new Date(q.last_review_time)) / (1000 * 60 * 60 * 24);
        return daysSinceReview >= getNextReviewInterval(q.review_count, 0);
    });

    // 按薄弱程度排序
    dueQuestions.sort((a, b) => {
        const aRate = getCategoryCorrectRate(a.category);
        const bRate = getCategoryCorrectRate(b.category);
        return aRate - bRate;
    });

    return dueQuestions.slice(0, limit);
}
```

## 七、学习流程

### 7.1 错题采集流程
1. 用户在软考网站做题
2. 浏览器插件检测到答题完成
3. 插件采集题目信息（题目、选项、答案、解析、分类）
4. 将数据发送到后端API
5. 后端存储数据并进行初步分析

### 7.2 学习复习流程
1. 用户登录系统
2. 系统根据复习周期和薄弱点，推荐待练习题目
3. 用户进行答题练习
4. 系统实时反馈并更新学习记录
5. 根据答题结果，更新下次复习时间
6. 生成学习报告

### 7.3 统计分析流程
1. 系统每日汇总学习数据
2. 计算各分类、章节的掌握度
3. 识别新的薄弱知识点
4. 生成统计报表
5. 提供学习建议
