一、项目总览与三层架构

数据采集层 — Chrome Extension MV3

从 ruankaodaren.com 抓取题目 DOM、选项、答案、解析、练习结果。content.js 注入页面采集，background.js 中转去重，popup 提供交互 UI。

数据处理层 — Flask + SQLite Backend

单文件 app.py（~6500 行），提供 100 条 RESTful API。UPSERT 入库，SRS 间隔重复算法，统一异常处理，按 IP 限流。

数据展示层 — React + Vite Frontend

21 条路由，25 个业务组件。无 UI 框架依赖，纯 CSS/SVG 实现图表。fetchAPI 统一封装，自动拼接 user_id。

十一、关键架构特征

双层去重与容错

dedupMap（内存+storage，TTL 5min）：isDuplicate 只读检查，markSent 成功后标记，unmarkSent 失败时清除。

pending_queue（storage，上限 500）：按 question_id 去重，逐条原子删除，retryCount + 死信移除，isProcessingQueue 互斥锁。

SRS 间隔重复算法

固定间隔序列：[1, 2, 4, 7, 15, 30, 60, 120] 天

wrong_questions.srs_stage → next_review_time → ReviewQueue 消费

flashcards.srs_stage → next_review_at → calculate_next_review()

UPSERT 数据合并

错题基于 question_id + user_id 去重，UPDATE 时用 COALESCE(NULLIF(?, ''), field) 补全空字段，wrong_count/error_count 累加。

练习会话基于 user_id + source_url + 5分钟窗口 去重。

章节聚合兼容查询

R25 后章节名存入 chapter 列，category 改为题型分类。所有分析接口使用 COALESCE(NULLIF(chapter, ''), category) 兼容新旧数据。

单文件后端架构

app.py ~6500 行，包含全部路由、建表、业务逻辑、种子数据。ensure_schema() 含动态 ALTER TABLE 兼容旧库迁移。

无图表库依赖

前端仅依赖 react + react-router-dom，图表（AbilityRadar、Statistics 等）使用纯 CSS/SVG 实现，体积小、可控性强。

UPSERT SQL 示例

-- 错题 UPSERT（基于 question_id + user_id）
UPDATE wrong_questions
SET correct_answer = COALESCE(NULLIF(?, ''), correct_answer),
    options = CASE WHEN ? NOT IN ('', '[]') AND (options IS NULL OR options = '[]')
                  THEN ? ELSE options END,
    analysis = COALESCE(NULLIF(?, ''), analysis),
    chapter = COALESCE(NULLIF(?, ''), chapter),
    wrong_count = wrong_count + 1,
    error_count = error_count + 1
WHERE question_id = ? AND user_id = ?

-- 练习会话 UPSERT（基于 user_id + source_url + 5分钟窗口）
SELECT id FROM practice_sessions
WHERE user_id = ? AND source_url = ?
  AND created_at >= datetime('now', '-5 minutes')
