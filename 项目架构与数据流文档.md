## 一、项目总览

```
ruankao-xtjgs/
├── backend/          # Flask + SQLite 后端（单文件 app.py，~6500 行，100 条 API 路由）
├── src/              # React + Vite 前端（25 个业务组件，21 条路由）
```

- **数据采集层**：Chrome 插件从 ruankaodaren.com 抓取题目 DOM、答案、解析、练习结果

- **数据展示层**：React 前端消费 API，渲染 21 个功能页面（错题库、统计、计划等）

### 4.4 离线队列容错流

```mermaid
flowchart TD
    START([发送失败/网络异常]) --> ADD[addToPendingQueue<br/>data + type + queue_id]
    ADD --> CHECK_LEN{队列长度 ≥ 500?}
    CHECK_LEN -->|是| DROP[丢弃最旧条目并告警]
    CHECK_LEN -->|否| DEDUP{同 question_id<br/>已在队列?}
    DEDUP -->|是| UPDATE_TS[更新 timestamp<br/>不重复插入]
    DEDUP -->|否| PUSH[push 到队列末尾]
    DROP --> STORAGE
    UPDATE_TS --> STORAGE
    PUSH --> STORAGE[(chrome.storage.local<br/>pending_queue)]

    ALARM[chrome.alarms<br/>每 30 秒触发] --> PROCESS[processPendingQueue]
    RETRY[popup 重试按钮] --> PROCESS

    PROCESS --> LOCK{isProcessingQueue<br/>互斥锁?}
    LOCK -->|占用中| SKIP[跳过本次]
    LOCK -->|空闲| ITER[遍历队列]

    ITER --> ITEM{队列项类型}
    ITEM -->|question| SEND_Q[sendToBackend<br/>skipDedup=true<br/>skipQueueOnFail=true]
    ITEM -->|session| SEND_S[sendSessionToBackend<br/>skipQueueOnFail=true<br/>skipHistoryOnFail=true]

    SEND_Q --> RESULT{成功?}
    SEND_S --> RESULT

    RESULT -->|成功| RM[removeFromPendingQueueById<br/>逐条原子删除]
    RESULT -->|失败| INC[retryCount + 1<br/>updateQueueItem]
    INC --> LIMIT{retryCount ≥ 10?}
    LIMIT -->|是| DEAD[移除死信<br/>告警]
    LIMIT -->|否| KEEP[保留队列<br/>等待下次重试]

    RM --> DONE([处理完成])
    DEAD --> DONE
    KEEP --> DONE

    style START fill:#ffebee
    style STORAGE fill:#fce4ec
    style DONE fill:#e8f5e9
    style DEAD fill:#fff3e0
```

## 六、关键架构特征

### 6.1 去重与容错机制

```
双层容错：
├── dedupMap（内存 + storage，TTL 5min）
│   ├── isDuplicate() 只读检查
│   ├── markSent() 发送成功后标记
│   └── unmarkSent() 失败时清除
└── pending_queue（chrome.storage.local，上限 500）
    ├── addToPendingQueue 按 question_id 去重
    ├── processPendingQueue 逐条原子删除（按 queue_id）
    ├── retryCount + 死信移除（MAX_ITEM_RETRIES=10）
    └── isProcessingQueue 互斥锁（防 alarm+retry 并发）
```

### 6.2 SRS 间隔重复算法

```
SRS_INTERVALS = [1, 2, 4, 7, 15, 30, 60, 120]  # 天

wrong_questions:
  srs_stage → next_review_time → ReviewQueue 消费

flashcards:
  srs_stage → next_review_at → calculate_next_review()
```

### 6.3 UPSERT 数据合并策略

```sql
-- 错题 UPSERT（基于 question_id + user_id）
UPDATE wrong_questions
SET correct_answer = COALESCE(NULLIF(?, ''), correct_answer),
    options = CASE WHEN ? NOT IN ('', '[]') AND (options IS NULL OR options = '[]')
                  THEN ? ELSE options END,
    analysis = COALESCE(NULLIF(?, ''), analysis),
    chapter = COALESCE(NULLIF(?, ''), chapter),
    wrong_count = wrong_count + 1,
    error_count = error_count + 1
WHERE question_id = ? AND user_id = ?

-- 练习会话 UPSERT（基于 user_id + source_url + 5分钟窗口）
SELECT id FROM practice_sessions
WHERE user_id = ? AND source_url = ?
  AND created_at >= datetime('now', '-5 minutes')
```

### 6.4 章节聚合兼容查询

R25 后章节名存入 `chapter` 列，`category` 改为题型分类。所有分析接口使用 `COALESCE(NULLIF(chapter, ''), category)` 兼容新旧数据。
